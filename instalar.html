Instalación del modulo

A continuación describiremos el proceso de instalación del plugin para su uso en Drupal.

Es necesario subir los archivos del plugin a la carpeta de modulos de Drupal. Esta carpeta se encuentra en:
[Drupal_Dir]/web/modules/

Paso 1 - Instalar el plugin en el Backoffice de Drupal

Para que el código funcione primero se necesita instalar y activar el proyecto desde el panel de administración, en la sección de modulos.

Paso 2 - Crear tabla en la base de datos

Ahora con el plugin reconocido por Drupal podemos proceder a crear la tabla para persistencia de datos del asistente.

Para ello solo hay que dirigirse a la URL: [Drupal_URL]/pm/install, y la base de datos habrá sido creada.

Paso 3 - Crear el archivo de configuración

Dentro de los archivos del plugin se encuentra el archivo pmconfig.yml.sample, hay que renombrarlo a pmconfig.yml, y en el escribiremos los nombres de maquina de Drupal para el contenido relacionado con el asistente.

Paso 4 - Crear la taxonomia "field_plugin"

Se requiere una taxonomia para poder usar los subplugins del Assistantmanager.

El nombre de maquina de la taxonomia debe ser "field_plugin" u otro, pero debe anotarse en el archivo pmconfig.yml en el parametro plugin_taxonomy.

Paso 5 - Añadir terminos para subplugins

Es necesario crear un termino en la taxonomia anterior por cada directorío dentro de Assistantmanager/pm/, se debe usar exactamente el mismo nombre, ya que este es el nombre identificativo de cada subplugin.

Paso #6 - Crear tipos de contenido para Asistentes y para pasos

Se debe crear un tipo de contenido para asistentes y otro para pasos, y los nombres de maquina de estos deben anotarse en el archivo de configuración del asistente.

Para poder enlazar pasos a asistentes se usa el plugin Entity Form.

Dentro de los pasos debe configurarse un input tipo select que liste los elementos dentro de la taxonomía creada para los subplugins.

Paso #7 - Empezar a crear Asistentes

Ya se pueden empezar a crear asistentes con los pasos requeridos y los plugins adecuados para ellos.

Un mismo plugin se puede reutilizar en múltiples pasos si fuera necesario, pero se recomienda crear pasos independientes para cada asistente.
