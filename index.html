Assistant Manager

Documentacion para modulo construido en Drupal



Introducción

El modulo Assistant Manager para Drupal permite crear y gestionar múltiples asistentes desde el backoffice para usarse para diferentes propósitos de forma flexible.

Los asistentes se crean como un tipo de contenido de Drupal, y a cada asistente se le asignan pasos, que son otro tipo de contenido.

Para poder enlazar pasos a un asistente se usa el plugin Entity Form (el cual debe configurarse adecuadamente o este no funcionara). Con este plugin se pueden poner los pasos deseados y se puede establecer un orden para ellos de forma manual.

Cada paso de asistente cuenta con un selector de subplugins, los cuales están definidos como una taxonomía de Drupal.

A su vez cada subplugin representa una carpeta de código dentro de la carpeta del Asistant Manager.

El poder del asistente esta en los subplugins, cada uno define un formulario de configuración a mostrar en el backoffice y con esta configuración genera un formulario que se le mostrara al usuario de la app web.

Cada elemento del formulario se puede personalizar para poder crear todo tipo de elementos UI que se pudieran necesitar en el proyecto.

Al final el usuario accederá a la URL del asistente y vera el formulario definido como primer paso, y conforme vaya introduciendo y mandando los datos que se le solicitan el asistente ira generando los formularios de los demás pasos.

El usuario incluso puede abandonar el asistente y después continuarlo desde donde lo dejo, ya que este cuenta con un sistema para almacenar los datos introducidos.
