Estructura del código que gestiona la API de GV

A continuación explicaremos los archivos relacionados con la implementación de la API de GV

src/
|--Assistant/
|  |--API.php
|  |--APIHelper.php
|  |--APIHotelHelper.php
|  |--APIHotelListado.php
|  |--APIHotelPresenter.php
|  |--Helper.php
|  |--Intereses.php
|  |--SessionControl.php

src/Assistant/API.php

Clase con todos los métodos para consumir la API.

src/Assistant/APIHelper.php

Clase que encapsula métodos auxiliares para procesar datos antes o después de enviarlos a la API.

src/Assistant/APIHotelHelper.php

Clase auxiliar para dar formato a arreglos que se enviarán o que se reciben de la API.

Cuenta con métodos para preparar y dar formato al filtro de fecha y de pasajeros según habitaciones para poder así hacer la búsqueda.

Otros métodos sirven para solicita a la API datos como las zonas, tipos y subtipos y ordenarlos de forma que se puedan relacionar con el listado de hoteles resultante y poder mostrar todos los datos relacionados con hoteles.

src/Assistant/APIHotelListado.php

Clase planeada para generar listado de Hoteles según los filtros del usuario.

src/Assistant/APIHotelPresenter.php

Clase intermedia para preparar el listado de hoteles para que pueda ser mostrado por el front.

src/Assistant/Helper.php

Clase con Helpers mas generales, cuenta por ejemplo con el método log("string") para poder imprimir datos desde cualquier parte del asistente hacia un archivo en sites/default/files/assitant/log.txt.

Otros métodos que aquí se encuentran son los que permiten permutar arrays.

src/Assistant/Intereses.php

Clase que permite gestionar el listado de Intereses marcados por el usuario, ayuda a que el asistente de Intereses pueda determinar a donde mandar al usuario al entrar por primera vez o después de volver tras abandonar.

src/Assistant/SessionControl.php

Clase con la lógica para gestionar la sesión interna y la del endpoint, cuenta con los métodos para inicializar, reiniciar o validarla.
